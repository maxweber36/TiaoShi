## 目标
- 在首页手动输入地址后，调用地址解析服务（高德地理编码），将地址转换为经纬度并写入全局位置。
- 替换占位提示，形成完整的“地址 → 坐标 → 展示”闭环。

## 改动范围
1. 新增 Hook 方法（src/hooks/useLocation.ts）
- geocodeAddress(address: string): Promise<Location>
  - 设置 `isLocating` 开始、清空错误；
  - 调用 `getCoordinatesFromAddress(address)`（已有实现：src/services/location.ts:162）；
  - 将返回的 `Location`（含 `address`）写入 `setLocation`；
  - 结束后复位 `isLocating`，并返回解析结果。

2. 接入首页（src/pages/Home.tsx）
- 在 `handleManualLocation` 中：
  - 使用 `geocodeAddress(manualAddress)` 替换占位 `alert`；
  - 成功后页面“当前位置”卡片显示解析得到的地址（已存在 UI）；
  - 保持现有 loading 状态与错误提示（`isSearching`/catch 提示）。

## 错误与边界处理
- 空地址：保留现有“请输入地址”提示并直接返回。
- 未配置密钥：`getCoordinatesFromAddress` 抛错，`handleManualLocation` catch 分支提示“地址解析失败”。
- API 失败或无结果：抛出并提示，不修改现有自动定位/IP定位逻辑。

## 配置
- 必填环境变量：`VITE_AMAP_API_KEY`
- 可选：`VITE_AMAP_SIG_SECRET`（启用请求签名，已由 `location.ts` 内置 `md5` 计算）。
