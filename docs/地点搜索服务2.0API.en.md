# Place Search Service 2.0 API

## Product Overview

The Place Search Service 2.0 is a Web API service that provides various place search capabilities, including keyword search, nearby search, polygon area search, and ID search.

Currently, the search does not support returning full data. Pagination queries support a maximum of 200 data records per request.

## Usage Instructions

## Keyword Search

#### Keyword Search API Service Address

| URL | Request Method |
|---|---|
| https://restapi.amap.com/v5/place/text?parameters | GET |

Parameters include required and optional parameters. All parameters are separated by ampersands (&). The following list enumerates these parameters and their usage rules.

#### Request Parameters

| Parameter Name | Meaning | Rule Description | Required | Default Value |
|---|---|---|---|---|
| key | Amap Key | User applies for Web Service API type Key on Amap official website | Required | None |
| keywords | Place Keywords | Text information of places to be retrieved. Only one keyword is supported, total text length cannot exceed 80 characters | Required (either keywords or types) | None |
| types | Specify Place Type | Place text search interface supports limiting search results by set POI types. Place type and poi typecode are similar content. Multiple poi typecodes can be passed, separated by "|". Content can refer to POI classification code table. Place (POI) list sorting is based on comprehensive weight sorting of Amap search capability. | Optional (either keywords or types) | 120000 (Commercial Residential)<br>150000 (Traffic Facility Services) |
| region | Search Administrative Division | Increase data recall weight in designated area. For strict limitation of recalled data in the area, please use city_limit parameter together. Can input citycode, adcode, cityname. Cityname only supports city level and Chinese, such as "北京市". | Optional | None, search nationwide by default |
| city_limit | Designated City Data Recall Limit | Optional values: true/false. When true, only recall data corresponding to the region area. | Optional | false |
| show_fields | Return Result Control | show_fields is used to filter optional fields in response results. Usage of show_fields must follow these rules: 1. Specific returnable field types refer to field types within "show_fields" in return result description below; 2. Multiple fields are separated by ","; 3. When show_fields is not set, only return fields within basic information class. | Optional | Empty |
| page_size | Current Page Display Data Count | page_size value range 1-25 | Optional | page_size defaults to 10 |
| page_num | Requested Page Number | Requested page number | Optional | page_num defaults to 1 |
| sig | Digital Signature | Please refer to Digital Signature Acquisition and Usage Method | Optional | None |
| output | Return Result Format Type | Default format is json, currently only supports json format. | Optional | json |
| callback | Callback Function | callback value is user-defined function name, this parameter is only valid when output parameter is set to JSON. | Optional | None |

#### Service Example

```
https://restapi.amap.com/v5/place/text?keywords=北京大学&types=141201&region=北京市&key=<用户的key>
```

| Parameter | Value | Remarks | Required |
|---|---|---|---|
| keywords | | Place keywords, text information of places to be retrieved. Only one keyword is supported, total text length cannot exceed 80 characters | keyword or types required |
| types | | Specify place type, place text search interface supports limiting search results by set POI type; place type and poi typecode are similar content, can pass multiple poi typecodes, separated by "|". Content can refer to POI classification code table; place (POI) list sorting is based on comprehensive weight sorting of Amap search capability | keyword or types required |
| region | | Search administrative division, increase data recall weight in designated area, for strict limitation of recalled data in area, please use city_limit parameter together. Can input citycode, adcode, cityname. Cityname only supports city level and Chinese, such as "北京市". | Optional |

#### Return Results

| Name | Type | Description |
|---|---|---|
| status | string | This API access status, returns 1 if successful, returns 0 if failed. |
| info | string | Description of access status value, returns "ok" if successful, returns error reason if failed. |
| infocode | string | Return status description, 10000 represents correct. |
| count | string | Actual number of poi points returned per single request |
| pois | object | Complete set of returned pois |
| poi | object | Complete return data contained in a single poi |
| name | string | poi name |
| id | string | poi unique identifier |
| location | string | poi longitude and latitude |
| type | string | poi type |
| typecode | string | poi classification code |
| pname | string | poi province |
| cityname | string | poi city |
| adname | string | poi district |
| address | string | poi detailed address |
| pcode | string | poi province code |
| adcode | string | poi area code |
| citycode | string | poi city code |

**Note**: The following fields need to be set through "show_fields" parameter for return.

| children | object | Return child POI information when set |
| business | object | Return poi business information when set |
| indoor | object | Return indoor related information when set |
| navi | object | Return navigation location related information when set |
| photos | object | Return poi picture related information when set |

## Nearby Search

#### Nearby Search API Service Address

| URL | Request Method |
|---|---|
| https://restapi.amap.com/v5/place/around?parameters | GET |

Parameters include required and optional parameters. All parameters are separated by ampersands (&). The following list enumerates these parameters and their usage rules.

#### Request Parameters

| Parameter Name | Meaning | Rule Description | Required | Default Value |
|---|---|---|---|---|
| key | Amap Key | User applies for Web Service API type Key on Amap official website | Required | None |
| keywords | Place Keywords | Text information of places to be retrieved. Only one keyword is supported, total text length cannot exceed 80 characters | Optional | None |
| types | Specify Place Type | Place text search interface supports limiting search results by set POI types. Place type and poi typecode are similar content. Multiple poi typecodes can be passed, separated by "|". Content can refer to POI classification code table. Place (POI) list sorting is based on comprehensive weight sorting of Amap search capability. When both keywords and types are empty, default types are 050000 (Catering Services), 070000 (Life Services), 120000 (Commercial Residential). | Optional | 050000 (Catering Services)<br>070000 (Life Services)<br>120000 (Commercial Residential) |
| location | Center Point Coordinates | Circular area retrieval center point, does not support multiple points. Longitude and latitude are separated by ",", longitude first, latitude second, longitude and latitude decimal places cannot exceed 6 digits | Required | None |
| radius | Search Radius | Value range: 0-50000, values greater than 50000 use default value, unit: meters | Optional | 5000 |
| sortrule | Sorting Rule | Specify sorting rule for return results. Sort by distance: distance; comprehensive sorting: weight | Optional | distance |
| region | Search Administrative Division | Increase data recall weight in designated area, for strict limitation of recalled data in area, please use city_limit parameter together. Can input administrative division name or corresponding citycode or adcode | Optional | None, search nationwide by default |
| city_limit | Designated City Data Recall Limit | Optional values: true/false. When true, only recall data corresponding to the region area | Optional | false |
| show_fields | Return Result Control | show_fields is used to filter optional fields in response results. | Optional | Empty |
| page_size | Current Page Display Data Count | page_size value range 1-25 | Optional | page_size defaults to 10 |
| page_num | Requested Page Number | Requested page number | Optional | page_num defaults to 1 |
| sig | Digital Signature | Please refer to Digital Signature Acquisition and Usage Method | Optional | None |
| output | Return Result Format Type | Default format is json, currently only supports json format | Optional | json |
| callback | Callback Function | callback value is user-defined function name, this parameter is only valid when output parameter is set to JSON | Optional | None |

#### Service Example

```
https://restapi.amap.com/v5/place/around?location=116.473168,39.993015&radius=10000&types=011100&key=<用户的key>
```

| Parameter | Value | Remarks | Required |
|---|---|---|---|
| location | | Center point coordinates, circular area retrieval center point, does not support multiple points. Longitude and latitude are separated by ",", longitude first, latitude second, longitude and latitude decimal places cannot exceed 6 digits | Yes |
| radius | | Search radius, value range: 0-50000, values greater than 50000 use default value, unit: meters | Optional |
| types | | Specify place type, place text search interface supports limiting search results by set POI type; place type and poi typecode are similar content, can pass multiple poi typecodes, separated by "|". Content can refer to POI classification code table; place (POI) list sorting is based on comprehensive weight sorting of Amap search capability | Optional |

#### Return Results

Same structure as keyword search.
